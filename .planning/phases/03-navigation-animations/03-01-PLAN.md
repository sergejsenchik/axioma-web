3. **script.js**: Add smart header scroll handler inside `$(document).ready()`, AFTER the smooth scroll section and BEFORE the scroll-reveal section. Use this pattern:

```javascript
// Smart header: hide on scroll down, show on scroll up
var lastScrollY = 0;
var scrollThreshold = 50;

$(window).on('scroll', function() {
  var currentScrollY = window.pageYOffset;

  if (currentScrollY > scrollThreshold) {
    if (currentScrollY > lastScrollY) {
      // Scrolling down -- hide navbar
      $('.site-navbar').addClass('navbar-hidden');
    } else {
      // Scrolling up -- show navbar
      $('.site-navbar').removeClass('navbar-hidden');
    }
  } else {
    // At top of page -- always show
    $('.site-navbar').removeClass('navbar-hidden');
  }

  lastScrollY = currentScrollY;
});
```

Also add padding-top to the `<body>` or `.hero-section` to account for the fixed navbar height, OR since the hero is `height: 100vh` and the navbar overlays it with z-index, no padding is needed (navbar sits on top of the hero). Verify that the navbar does not push content down -- it should overlay, not displace.

Add these rules:

```css
/* Button text-swap hover animation (ANIM-03) */
.btn-text-in {
  transition: transform 0.35s ease, opacity 0.35s ease;
}

.btn-text-out {
  transition: transform 0.35s ease, opacity 0.35s ease;

/* Hover: text-in slides up and fades, text-out slides in from below */
.btn-cta:hover .btn-text-in,
.btn-cta-light:hover .btn-text-in,
.btn-cta-dark:hover .btn-text-in {
  transform: translateY(-120%);
  opacity: 0;
}

.btn-cta:hover .btn-text-out,
.btn-cta-light:hover .btn-text-out,
.btn-cta-dark:hover .btn-text-out {
  transform: translateY(0);
  opacity: 1;
}
```

Key per CONTEXT.md:
- Same text on both `.btn-text-in` and `.btn-text-out` -- this is a purely visual slide motion effect
- 0.35s ease timing (matches Webflow)
- Applied to all three CTA variants via compound selector
- The existing `.btn-text-out` base styles already have `position: absolute; transform: translateY(120%); opacity: 0;` -- just add the transition property to it
- Do NOT duplicate the existing `.btn-text-out` positioning -- only ADD the `transition` property to the existing rule

Verify the `.btn-text-in` transition does not conflict with the existing `.btn-text-wrap` styles. The `.btn-text-wrap` has `overflow: hidden` which is necessary for the text to clip as it slides out.
